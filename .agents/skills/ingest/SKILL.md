# ingest 技能

## 核心目标

把 `raw/` 中的原始资料**编译**进已有 wiki 知识结构，而不是按源文件批量生成新页面。

执行时遵守根目录 `AGENTS.md` 全部铁律，关键提醒：
- 铁律1：raw/ 只读
- 铁律3：新建页面前先按 `naming/SKILL.md` 确定文件名
- 铁律4：优先更新已有页面
- 铁律5：concepts/ 页面必须包含 6 个结构部分

## 触发逻辑

1. 用户执行 `/ingest`：扫描 `llm-wiki-ai/raw/` 下待处理文件
2. 用户执行 `/ingest <path>`：仅处理指定文件
3. 用户自然语言要求把资料加入知识库时，执行 ingest

## 编译流水线

### 步骤 1：先读取现有知识结构

读取原始资料前，先读：
- `llm-wiki-ai/index.md`
- 与当前资料主题最相关的已有 summary / concept / entity / synthesis 页面

目的是判断资料应编译进现有 wiki 的哪一部分，而不是从零生成一组新页面。

### 步骤 2：读取源文件

读取指定 `raw/` 文件，完整理解内容。若遇到无法直接解析的文件类型，先向用户说明，不要伪造内容。

### 步骤 3：提取知识元素

提取：主题、概念、实体、观点、方法、事实、案例

若资料属于已有主题簇，优先更新已有页面；确实是新主题时，才新建。

### 步骤 4：判定页面类型

写任何页面前，先判断知识对象属于哪一层（各类型定义见 `naming/SKILL.md`）：

1. 具体对象 → `wiki/entities/`
2. 抽象知识对象 → `wiki/concepts/`
3. 知识主题聚合 → `wiki/summaries/`
4. 问题/比较/策略/判断 → `wiki/synthesis/`

类型判断不清时，先不创建页面。

### 步骤 5：更新主题页

- `summaries/` 是"每个知识主题一页总结"，不是"每篇原文一页摘要"
- 若资料属于已有主题，更新对应 summary
- 若形成新主题，新建 summary（主题名必须抽象化，不得直接复用原文标题）

### 步骤 6：更新概念页和实体页

- 概念写入 `wiki/concepts/`，实体写入 `wiki/entities/`
- 已存在则增量合并，不存在则新建
- 发现明显冲突时，优先标注冲突来源，不静默覆盖
- 新建页面前，必须先按 `naming/SKILL.md` 确定文件名（铁律3）
- concepts/ 页面结构见铁律5

### 步骤 7：必要时更新综合页

资料带来问题、对比、策略、判断价值时，更新或创建 `wiki/synthesis/` 页面。不要把主题介绍误写成 synthesis。

### 步骤 8：补齐来源回指

所有新建或更新的知识页，必须显式回指相关 `raw/` 来源路径。不能产出脱离来源的纯结论页。

### 步骤 9：更新索引与日志（铁律2）

完成后必须：
- 更新 `llm-wiki-ai/index.md`
- 追加 `llm-wiki-ai/log.md`

日志格式：
```markdown
## [YYYY-MM-DD] Ingest | 标题
- 来源：`raw/...`
- 更新/新建页面列表
- 更新 `index.md`
```
