# 大模型微调工程实践

> 这个页面总结的是大模型微调的**工程全流程**：方法选型 → 环境搭建 → 数据准备 → 训练配置 → 保存部署 → 常见问题排查。原理层见 [[方法-LoRA]]。

## 知识核心

- **方法选型遵循"最小代价原则"**：LoRA 是默认起点；显存不够时换 QLoRA；任务极简且数据极少时才考虑 P-Tuning；全参数微调几乎不推荐（除非模型小且数据极丰富）
- **数据格式决定微调质量**：instruction-response 三元组（instruction / input / output）是标准格式；tokenize 时 prompt 部分的 label 设为 -100 以忽略损失，只对 output 部分计算 loss
- **显存优化三件套缺一不可**：fp16/bf16 精度 + gradient_checkpointing（重算激活值）+ gradient_accumulation_steps（小 batch 模拟大 batch）
- **LoRA 关键参数**：r=8 是通用起点，lora_alpha=32，dropout=0.1，target_modules 覆盖 Q/K/V/O + FFN（gate/up/down）；显存紧张时只取 Q/V
- **合并权重时机**：测试阶段用 AutoPeftModelForCausalLM 加载适配器（灵活）；生产部署用 merge_and_unload() 合并（脱离 PEFT 依赖）

## 完整工程流水线

### 1. 环境与硬件

| 模型规模 | LoRA（FP16） | QLoRA（4-bit） |
|---------|------------|--------------|
| 7B | 需 15 GB+ 显存 | 约 10-12 GB |
| 13B | 需 26 GB+ | 约 14-16 GB |
| 65B+ | 需多卡 | 单卡 48 GB 可行 |

核心依赖：`transformers` + `peft` + `datasets` + `bitsandbytes`（QLoRA 必须）+ `accelerate`

### 2. 数据准备

```json
{
  "instruction": "请写一首关于春天的五言绝句。",
  "input": "",
  "output": "春风拂面草木新，..."
}
```

- prompt = instruction + input（可选），output 是目标
- tokenize 后：prompt 对应位置 label=-100，output 对应位置 label=token_id
- 按 90/10 划分训练集/验证集，先用少量数据调通流程

### 3. 关键训练参数

```python
lora_config = LoraConfig(
    r=8,                  # rank：4-8 简单任务，16-64 复杂适配
    lora_alpha=32,        # 缩放因子，实际倍数 = alpha/r
    lora_dropout=0.1,     # 防过拟合，数据多时可设 0
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                    "gate_proj", "up_proj", "down_proj"]
)

training_args = TrainingArguments(
    per_device_train_batch_size=4,
    gradient_accumulation_steps=8,   # 有效 batch = 4×8 = 32
    learning_rate=2e-5,              # 比预训练小 10-100 倍
    fp16=True,
    gradient_checkpointing=True,     # 节省激活值显存
    num_train_epochs=3,
)
```

### 4. 保存与部署

```python
# 只保存 LoRA 适配器（几十 MB）
model.save_pretrained("./lora-adapter")

# 推理：PEFT 自动合并
from peft import AutoPeftModelForCausalLM
model = AutoPeftModelForCausalLM.from_pretrained("./lora-adapter")

# 生产部署：合并进模型（完整大小，脱离 PEFT）
merged = model.merge_and_unload()
merged.save_pretrained("./full-finetuned-model")
```

## 常见问题与排查

| 问题 | 优先排查方向 |
|------|------------|
| CUDA OOM | ① 减小 batch_size ② 开 gradient_checkpointing ③ 换 QLoRA 4-bit ④ 换小模型 |
| Loss 变 NaN / 梯度爆炸 | ① 设 max_grad_norm=1.0 ② 降低 learning_rate ③ 检查数据异常（超长序列/乱码） |
| Loss 下降停滞 | ① 增加 epoch + 早停监控验证集 ② 检查数据质量和格式 ③ 增大 r |
| PEFT 加载报错 | ① 升级 transformers/peft 版本 ② 手动指定 base_model_name_or_path |

## 适用边界

- **这套流程适合**：格式/风格定制，领域术语理解，1000-10 万条标注数据，单卡或少卡环境
- **不适合替代 RAG**：微调不能注入最新知识，知识截止问题仍然存在
- **数据量临界点**：< 1000 条优先考虑 Few-Shot；> 10 万条可以评估全参数微调的必要性
