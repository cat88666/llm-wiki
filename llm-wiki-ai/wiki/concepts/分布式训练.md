# 分布式训练

> 一句话定义：分布式训练通过数据并行（DP）、张量并行（TP）、流水线并行（PP）和 ZeRO 的组合，解决大模型无法在单张 GPU 上训练的显存和算力瓶颈。

## 核心要点
- **三种并行策略解决不同瓶颈**：DP 扩充训练吞吐量（多 GPU 处理更多 batch），TP 切分单层的大矩阵（解决单层放不下一张卡），PP 切分模型层数（解决层数太多放不下一张卡）
- **TP 通信量最大，必须用 NVLink**：TP 的 All-Reduce 发生在每个前向/反向传播的每一层，通信频率极高；节点内 NVLink 带宽约 600GB/s，跨节点 InfiniBand 约 100Gb/s，差距 50 倍以上；TP 跨节点几乎无法获得加速
- **PP 通信量小但有 bubble 问题**：流水线并行将模型层分段分配到不同 GPU，层间传递 activation；设备 bubble（等待时间）是 PP 效率的主要损耗，Interleaved Pipeline（1F1B 调度）可将 bubble 率从 (p-1)/p 降低
- **ZeRO 和并行策略正交互补**：ZeRO（Zero Redundancy Optimizer）消除数据并行中的参数/梯度/优化器状态冗余（ZeRO-1 只分片优化器状态，ZeRO-2 加上梯度，ZeRO-3 加上参数）；TP/PP 解决单层/单机放不下的问题，两者互不冲突
- **超大模型需要 3D 并行**：百亿参数以上模型通常需要 DP + TP + PP 三者组合（称为 3D 并行）；典型配置：TP=8（单机 8 卡），PP=8（8 个机器流水线），DP=N（数据并行扩充吞吐）

## 详细说明

数据并行（DP）的实现最简单：每个 GPU 持有完整模型副本，处理不同 batch 的数据，反向传播后 All-Reduce 同步梯度，更新参数。缺点是每张 GPU 都需要存储完整模型（包括参数 + 梯度 + 优化器状态），对于 70B 模型，仅 Adam 优化器状态就需要约 560GB（参数×4，fp32 存储）。

ZeRO 的三个阶段对应不同粒度的分片：ZeRO-1 只分片优化器状态（Adam 的 m/v 矩阵），通信量与 DP 相同，显存节省约 4x；ZeRO-2 再分片梯度，通信量增加约 1.5x；ZeRO-3 再分片参数，每次前向/反向需要 All-Gather 收集参数，通信量约是 ZeRO-1 的 3 倍，但显存节省最大（理论上 N 卡减少 N 倍）。

张量并行（TP/Megatron-LM 方式）的典型实现：将 Attention 的 Q/K/V 投影矩阵按列切分，每张 GPU 负责部分 Head 的计算；FFN 层按"列切分 → 行切分"方式切分两个线性层，前向只需一次 All-Reduce，后向也只需一次 All-Reduce。

## 与其他概念的关系
- 依赖 [[梯度下降与优化]]：分布式训练的核心是同步梯度（All-Reduce），ZeRO 分片的是优化器状态（Adam 的 m/v）
- 关联 [[多头注意力]]：TP 将注意力 Head 分配到不同 GPU，每个 GPU 负责部分 Head，是 TP 在注意力层的主要切分维度
- 关联 [[KV-Cache]]：推理阶段的 TP 要求 KV Cache 也按 Head 分片，每个 GPU 只保存自己负责的 Head 的 KV Cache

## 应用场景
- 7B-13B 参数模型：通常 ZeRO-2/3 + DP 即可，不需要 TP/PP
- 70B 参数模型：需要 TP=8（单机 8×A100）或 ZeRO-3；推理通常 TP=2/4
- 175B+ 参数模型（GPT-3 级别）：必须 3D 并行；训练时 TP=8、PP=8、DP=数十个机器节点
- 推理阶段：TP 常用于多 GPU 推理加速（TP=2/4），PP 在推理中因 bubble 问题不常用；vLLM 的 TP 推理是业界标准
