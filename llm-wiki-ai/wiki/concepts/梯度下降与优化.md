# 梯度下降与优化

> 一句话定义：梯度下降通过沿损失函数梯度的反方向迭代更新参数，是所有参数化机器学习模型的核心学习机制。

## 核心要点
- 三种梯度下降变体：批量梯度下降（全量数据，稳定但慢），随机梯度下降 SGD（单样本，噪声大），Mini-batch SGD（折中，深度学习标准实践，batch size 通常 32-512）
- 梯度消失/爆炸的根本原因：反向传播通过链式法则连乘，深层网络梯度 < 1 的连乘趋近于零（消失），梯度 > 1 的连乘趋近于无穷（爆炸）；解法分别是 ReLU 激活、残差连接、梯度裁剪（clip_grad_norm）
- Adam 的核心机制：一阶矩估计（动量，平滑梯度方向）+ 二阶矩估计（自适应学习率，对每个参数独立调节）；初始阶段有偏差，需要偏差修正（除以 1-β^t）
- AdamW = Adam + 解耦 Weight Decay：标准 Adam 的 L2 正则被自适应学习率削弱（正则化强度不均匀），AdamW 将权重衰减从梯度更新中分离，是 LLM 训练的主流选择
- 学习率调度是训练稳定性的关键：Warmup（线性增长到峰值）→ Cosine Decay（平滑下降）是大模型训练标准调度，Warmup 期的低学习率防止训练初期参数震荡

## 详细说明

梯度消失问题在 Transformer 中通过两种机制解决：一是残差连接（Residual Connection），梯度可以绕过任意子层直接传播到早期层；二是 Pre-LayerNorm，将归一化放在子层输入前，使梯度在残差路径上传播更平顺。这两个机制的组合使得 100+ 层的 Transformer 可以稳定训练。

Adam 的超参数设置在 LLM 训练中已有经验共识：β1=0.9（动量），β2=0.95（二阶矩，比默认 0.999 小，更快适应梯度变化），ε=1e-8（数值稳定项），Weight Decay=0.1。学习率峰值通常在 1e-4 到 3e-4 范围，batch size 和学习率需要线性缩放（batch 翻倍，学习率相应翻倍）。

梯度裁剪（Gradient Clipping）是 LLM 训练必备的稳定性机制：当梯度的全局 L2 范数超过阈值（通常 1.0）时，等比例缩小所有梯度，防止参数更新过大导致训练崩溃（loss spike）。

## 与其他概念的关系
- 关联 [[正则化]]：L2 正则等价于 Weight Decay，AdamW 将二者解耦；正则化修改损失函数，优化决定如何最小化它
- 关联 [[分布式训练]]：分布式训练中需要同步梯度（All-Reduce），ZeRO 通过分片优化器状态（Adam 的一阶/二阶矩）来节省显存
- 关联 [[Self-Attention]]：Transformer 的梯度需要通过注意力矩阵的 softmax 反向传播，FlashAttention 的 IO 优化也考虑了反向传播的显存效率

## 应用场景
- 深度学习模型训练（CNN/RNN/Transformer）：Mini-batch SGD/Adam 是标准选择
- LLM 预训练：AdamW + Cosine LR Schedule + Gradient Clipping 是工业标准配置
- 微调（Fine-tuning）：学习率通常比预训练小 10-100 倍，防止破坏预训练参数
- 强化学习（RLHF）：PPO 算法中梯度更新需要 clip ratio 限制策略变化幅度，是另一种梯度稳定性机制
