## 定义

Agent（智能体）是由大语言模型驱动、具备规划、记忆和工具使用能力的任务执行系统。它不只是一次性生成答案，而是能够围绕目标持续决策、调用工具、观察结果并继续推进。

## 核心组成

### 1. 规划
- 把复杂任务拆成可执行步骤
- 决定下一步应该调用什么能力

### 2. 记忆
- 短期记忆：当前对话与任务状态
- 长期记忆：外部知识、用户偏好、历史记录

### 3. 工具使用
- 搜索、代码执行、数据库查询
- 业务系统调用、插件、MCP 工具

## 与 Function Calling 的关系
- Function Calling 更偏单次结构化工具调用
- Agent 更偏多步任务编排与状态推进
- MCP 让工具协议标准化，便于 Agent 跨系统复用能力

## 适用场景
- 多步任务执行
- 业务流程自动化
- 智能运维、客服工单、数据分析流水线
- 需要持续观察与修正的复杂任务

## 风险与约束
- 任务链路越长，越需要状态管理与可观测性
- 工具描述不清会导致错误调用
- 必须做好权限控制、失败回退和审计
