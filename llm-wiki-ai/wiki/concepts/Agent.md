# Agent

> 一句话定义：Agent 的本质，是让模型不只生成一次答案，而是围绕目标持续进行“决策 - 行动 - 观察 - 再决策”的闭环。

## 第一性原理
- 很多真实任务不是一次回答就能完成，而是需要多步推进、调用外部能力并根据结果调整路径。
- 单轮问答模型只解决“给出一个响应”，Agent 解决的是“把事情做完”。
- 所以 Agent 的根问题不是更会说，而是能不能把语言理解转成持续执行。

## 核心机制
- 目标分解：把复杂任务拆成可执行步骤。
- 工具调用：通过外部系统获取信息或执行动作。
- 状态维护：保存上下文、任务进度、约束和中间结果。
- 反馈闭环：根据工具结果、环境变化或失败信号重新规划下一步。

## 关键权衡
- 自主性越强，覆盖任务越广，但不确定性和失控风险也越高。
- 工具越多，能力越强，但决策空间更大，错误路由概率也更高。
- Agent 的难点通常不在“写一个 loop”，而在状态设计、失败恢复、权限边界和观察反馈质量。

## 与其他概念的关系
- [[Function-Calling]]：是 Agent 执行动作的基础接口。
- [[Context_Engineering]]：决定 Agent 在每一步能看到什么上下文和约束。
- [[RAG]]：可以作为 Agent 的一个知识获取子能力。

## 应用边界
- 适合需要多步执行和环境交互的任务。
- 对一次性、确定性的简单任务，用 Agent 往往是过度设计。
