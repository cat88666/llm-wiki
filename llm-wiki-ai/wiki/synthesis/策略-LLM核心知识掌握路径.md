# 策略：LLM核心知识掌握路径

> 核心原则：LLM 知识不应该是记忆清单，而是建立**判断力**——能解释为什么、能预判边界、能做出选型决策。

## 学习顺序总览

```
阶段1（地基）：Transformer 架构原理      → 理解 LLM 的运行基础
阶段2（机制）：LLM 核心运行机制          → 理解推理/成本/限制的根源
阶段3（使用）：Prompt 工程               → 有效使用 LLM 的基础技能
阶段4（系统）：RAG + Agent              → 构建实际应用能力
阶段5（生产）：评估与生产化              → 让系统稳定可控
```

机器学习基础（监督学习、梯度下降、正则化）是可选前置。不做 ML 工程师可以轻量了解，不必深究。

---

## 阶段1：Transformer 架构原理（优先级：必须）

**目标**：理解 LLM 的设计基础，建立解释性思维。

**核心要掌握的判断力**：
- Self-Attention 的本质是什么（动态权重，不是固定卷积）
- 为什么除以 √d_k（数值稳定，不是魔法数字）
- 多头注意力的价值是什么（子空间多样性，非简单叠加）
- O(n²) 复杂度意味着什么（长文本的根本瓶颈）
- Pre-LN vs Post-LN 哪个是现代标准（Pre-LN，为什么）

**不需要深究**：反向传播数学推导、注意力公式记忆、位置编码各种变体细节。

**Wiki 阅读顺序**：
1. [[主题-Transformer架构原理]] — 全景，先建立整体认知
2. [[机制-Self-Attention]] — 核心计算单元
3. [[机制-多头注意力]] — 子空间扩展

## 阶段2：LLM 核心运行机制（优先级：必须）

**目标**：理解从"Transformer"到"实际LLM系统"的工程现实。

**核心要掌握的判断力**：
- KV Cache 是什么，为什么是显存瓶颈（推理的核心成本来源）
- Tokenizer 如何影响成本和上下文利用率（选型参考）
- FlashAttention 解决的是什么问题（IO 不是数学）
- 分布式训练4种策略适用什么场景（不是为了实现，是为了选型）

**不需要深究**：具体的 CUDA 实现、Megatron/DeepSpeed 代码细节。

1. [[主题-LLM核心机制与推理优化]] — 主题全览
2. [[机制-KV-Cache]] — 推理成本核心
3. [[概念-Tokenizer]] — 影响成本和上下文的隐形因素
4. [[机制-FlashAttention]] — 工程优化理解

## 阶段3：Prompt 工程（优先级：必须，立刻见效）

**目标**：建立系统化使用 LLM 的工程实践，不再靠直觉写提示词。

**核心要掌握的判断力**：
- Prompt 的本质是什么（收缩输出空间，而非命令模型）
- temperature 参数如何按任务分化（0-0.3 确定，0.7+ 创作）
- 示例（Few-shot）为什么优于规则描述
- CoT 适合什么任务（推理密集），不适合什么（简单查询）
- 上下文工程（Context Engineering）与 Prompt 工程的区别

## 阶段4：RAG + Agent（优先级：高，核心应用能力）

**目标**：掌握 LLM 应用的两条最主要工程路径。

### RAG（知识增强）
**核心要掌握的判断力**：
- RAG 解决什么问题（知识截止/幻觉），不解决什么（模型行为问题）
- 检索链路的决策顺序（分块→向量→混合检索→Reranker→Prompt组装）
- Reranker 是 ROI 最高的单点优化（为什么）
- 混合检索为什么优于纯向量（精确名词匹配的失效场景）

**Wiki 阅读顺序**：
1. [[概念-RAG]] — 完整链路理解
2. [[方法-混合检索]] — 检索策略提升
3. [[主题-RAG系统设计]] — 工程全链路
4. [[对比-RAG-vs-微调-vs-Agent]] — 三条路径的选型判断

### Agent（行动扩展）
**核心要掌握的判断力**：
- Agent 解决什么问题（多步执行，不是更聪明的问答）
- 单 Agent 什么时候足够（工具≤10，步骤≤5）
- 工具描述是最重要的工程点
- 失败处理比规划设计更重要

**Wiki 阅读顺序**：
1. [[概念-Agent]] — 本质理解
2. [[概念-Function-Calling]] — 工具接口设计
3. [[主题-Agent与工具调用]] — 工程实践

## 阶段5：评估与生产化（优先级：高，上线前必须）

**目标**：知道如何让 LLM 系统在生产中可测、可控、可维护。

**核心要掌握的判断力**：
- 最小可观测性追踪集包含哪些字段
- RAGAS 四指标的含义，Faithfulness 为什么最重要
- 成本控制的三个主要手段（路由/缓存/动态注入）
- 上线前 8 项检查清单（不可跳过）
