## 概述

大语言模型从零开始训练，需要经历多个不同阶段，每个阶段使用不同类型的数据、不同的训练目标，针对模型的不同能力进行强化。这套训练范式以 GPT 系列为代表，已成为当前绝大多数（约 80% 以上）大语言模型所遵循的基本框架。

整个过程可以拆分为两大部分：

- **预训练（Pre-training）**：使用海量无标注文本，让模型学习人类语言的规律与知识；
- **后训练（Post-training）**：依次经历有监督微调（SFT）、奖励模型训练（Reward Model）和强化学习（RL），使模型能够执行具体任务并与人类期望对齐。

---

## 核心概念：监督学习与自监督学习

理解训练范式，首先需要区分几个关键术语。

**有监督学习（Supervised Learning）**指使用带有人工或 AI 标注标签（即正确答案）的数据进行训练。例如：情感分类中"正向/负向"标签、句子相似度标注等，均属有监督数据。

**自监督学习（Self-supervised Learning）**指正确答案直接来自输入数据本身，无需人工标注。预训练阶段就属于自监督：给模型前 $n$ 个字，让它预测第 $n+1$ 个字，而第 $n+1$ 个字本身就存在于原始训练数据中，既是输入也是标签。

**无监督学习（Unsupervised Learning）**指完全没有标签的数据。预训练有时也被称为无监督，与自监督含义在工程实践中接近，通用。

**半监督学习（Semi-supervised Learning）**则是先做自监督（预训练），再做有监督（SFT），两个阶段组合的模式，也是当前大语言模型训练的整体结构。

---

## 第一阶段：预训练（Pre-training）

### 定义与目的

预训练是整个大语言模型训练中**最耗算力、最花钱、也是耗时最长**的阶段。其核心目标是让模型通过阅读海量人类文字，学习语言的规律和内在知识。类比而言，这相当于让一个刚出生的孩子持续不断地阅读人类写作的所有文献，积累对语言的感知与理解。

### 数据规模

| 模型 | 训练数据量 | 模型参数量 | 使用硬件 | 训练时长 |
|------|-----------|-----------|---------|---------|
| LLaMA 2 | 1.4T tokens | 65B（650亿） | 2000+ 张 A100（约6–10万元/张） | 约 20 天 |
| LLaMA 3 | 15T tokens | 405B（4050亿） | 16000+ 张 H100（约20万元/张） | 约 50 天 |
| 当前主流模型 | 10–40T tokens | 不等 | — | — |

单位说明：M = 百万（$10^6$），B = 十亿（$10^9$），T = 万亿（$10^{12}$）。1 token 约等于 0.7–0.8 个汉字。15T tokens 大致对应 10–13 万亿个汉字。

### 核心训练逻辑：段落联合概率

预训练阶段使用的训练目标是**段落联合概率（Joint Probability of a Paragraph）**。

其直觉如下：给定一段训练文本，从第一个字开始，每次给模型前 $t$ 个字，让它预测第 $t+1$ 个字。模型会输出词表中所有候选词的概率分布（词表可能包含数万个 token），训练代码只取出正确答案对应的那个概率值。将整段文本所有位置对应的概率值连乘，即得到该段落的联合概率：

$$P(\text{段落}) = \prod_{t=1}^{T} P(w_t \mid w_1, w_2, \ldots, w_{t-1})$$

该概率越大，说明模型对这段文字的预测能力越强，即越"聪明"。刚初始化的模型接近随机猜测，每个 token 的概率约为 $1/|\mathcal{V}|$（$|\mathcal{V}|$ 为词表大小），联合概率极小；随着训练推进，高频搭配词的概率显著提升，联合概率随之增大。

训练流程如下：

1. 将数十亿条训练数据按**批次（batch）**（如 100 万条/批）喂给模型；
2. 每个批次计算所有训练样本的段落联合概率；
3. 以提升平均联合概率为目标，通过反向传播更新模型参数；
4. 循环至所有数据过完一轮，称为一个 **epoch**；
5. 预训练阶段通常进行 **2–3 个 epoch** 即结束。

### 预训练结束后的模型特性

预训练结束后，模型在语言能力和知识储备方面已与人类接近，能够流畅地"接话茬"——即根据上文进行续写。但它**缺乏任务执行能力**：面对具体的问答或指令时，往往倾向于将提示词当作文章段落继续补全，而非给出针对性的回答。这是进入后训练阶段的原因。

## 第二阶段：有监督微调（Supervised Fine-Tuning, SFT）

### 定义与目的

SFT（全称 Supervised Fine-Tuning）是后训练的第一步。其目的是在预训练模型的基础上，教会模型**执行具体任务**，尤其是**指令遵循（Instruction Following）**——即让模型理解"用户让它做什么"并给出对应的回答，而不是泛泛续写。

开源社区中，模型名称后缀带有 **Instruct** 的（如 Qwen-72B-Instruct），通常表示已经过高质量 SFT 微调；未加后缀的为基座模型，SFT 能力相对较弱。

### 训练数据

SFT 阶段使用**一问一答（Q&A）格式**的标注数据，数据量远少于预训练，但质量要求更高。

| 模型 | 预训练数据量 | SFT 数据量 |
|------|-----------|-----------|
| GPT-3 | 约 7000 万条 | 约 1.3 万条 |
| LLaMA 3 | 数十亿条 | 数十万至 100 万条 |

总体而言，SFT 数据量比预训练少约 **5000 到 10000 倍**，但对每一条数据的质量（准确性、深度、格式规范性）要求极高。早期 OpenAI 为 GPT 的 SFT 阶段聘请了 80 位不同领域博士手动撰写训练答案；现在越来越多由 AI 参与生成和标注。

### 训练逻辑

SFT 阶段与预训练的训练目标完全相同——最大化段落联合概率——但处理方式略有不同：

- 预训练：从第 1 个字开始逐步预测，问题与答案不做区分；
- SFT：将问题部分整体作为前缀（提示词），从第一个答案 token 开始计算联合概率，只对答案部分的生成进行优化。

这样模型学会的是"看到这类问题，应该生成这类回答"，而非单纯地续写文字。

## 第三阶段：后训练三部曲（Reward Model + 强化学习）

SFT 之后，模型已具备基本的任务执行能力，但其输出质量往往还不够令人满意——可能回答泛泛而谈、深度不足、或不符合人类价值观。这一过程称为**对齐（Alignment）**：使模型的输出与人类期望、标准和价值观保持一致。

GPT 使用的经典后训练方案分为三个步骤，也被称为 **RLHF（Reinforcement Learning from Human Feedback）**：

### 步骤一：SFT（高质量有监督微调）

同前述 SFT 阶段，使用人工或 AI 撰写的高质量问答对进行初步微调，使模型具备指令遵循能力。

### 步骤二：训练奖励模型（Reward Model, RM）

#### 偏好数据的收集

从 SFT 模型对同一个问题生成**多个（通常 4 个）不同回答**，由人类标注工程师对这些回答进行**排序（ranking）**，而非直接打分（直接打分容易因标注员标准不统一导致数据质量低下）。

OpenAI 的做法：每个问题生成 4 个回答（A、B、C、D），标注员给出排序（如 D > C > B = A）。4 个回答排序相当于一次性标注了 $\binom{4}{2} = 6$ 条偏好数据，标注效率大幅提升。

这类数据统称为**偏好数据（Preference Data）**，记录的是人类在不同回答之间的偏好关系。

#### 奖励模型的训练

奖励模型（RM）**从 SFT 模型分支**出来，通过在偏好数据上训练，学习"模拟人类评判标准"的能力——输入一个问题和一个回答，输出一个分数（0–100）。

RM 的参数量不必与当前最强模型相同，可以选择较小的版本（如用 32B 的 SFT 模型训练 RM，而最强模型为 500B），由对比实验决定最优选择。

RM 的核心作用：替代人类持续为大语言模型的输出打分，从而使强化学习可以自动化运行，而无需每次都依赖人工评估。

### 步骤三：强化学习（PPO）

强化学习阶段的目标是：基于 RM 的打分信号，不断优化当前最强大语言模型，使其生成的回答得分更高，即更符合人类偏好。

**核心流程：**

1. 准备数十万条提示词（与 SFT 和 RM 阶段的提示词高度重叠）；
2. 让当前最强大语言模型（即另一个 SFT 模型分支）对每条提示词生成回答；
3. 让 RM 对每条回答打分，得到数十万个分数；
4. 以**最大化综合得分**为训练目标，通过 **PPO（Proximal Policy Optimization）** 算法更新大语言模型内部参数；
5. 如此循环，模型能力持续提升。

**探索与遵循（Exploration vs. Exploitation）：** 强化学习的关键特性。大语言模型每次生成 token 时并非总选概率最高的那个（遵循），而是保留一定随机性（探索），以发现更优的生成路径。RM 的打分为探索提供了反馈信号。

#### RM 的周期性重训

随着强化学习不断推进，大语言模型的能力会超出当前 RM 的评判范围——模型生成的高质量回答，可能是 RM 训练时从未见过的水平。此时需要**暂停强化学习，重新收集偏好数据、重训 RM**，然后再继续强化学习。这一"运动员—教练员"的迭代机制是 GPT 训练范式的核心循环。

类比：运动员从校队进省队，省队教练不够用了，再换国家队教练；不断升级教练的标准，运动员的水平才能持续提升。

**损失函数补充：** PPO 阶段的损失函数并非单纯最大化 RM 得分，还融入了预训练和 SFT 阶段的段落联合概率约束，防止模型在追求高分的过程中产生过度偏移或退化。

## 开源生态的差异：LLaMA 的训练范式

LLaMA 系列（Meta 开源模型）在某一时期是开源生态中最强的大语言模型，但其训练范式与 GPT 有所不同。以 LLaMA 3 为代表，其区别主要体现在：

### 与 GPT 范式的对比

| 维度 | GPT 范式 | LLaMA 范式 |
|------|---------|-----------|
| 偏好数据收集 | 一问题生成 4 个回答，标排序 | 一问题生成 2 个回答，标排序及程度（"显著好""轻微好"等） |
| RM 的用途 | 用于强化学习打分 | 用于 **Rejection Sampling** 数据过滤，筛选高质量 SFT 数据 |
| 后训练对齐方法 | PPO（强化学习） | **DPO（Direct Preference Optimization）** |
| 循环次数 | 多轮循环（次数视实验而定） | 大约循环 **6 轮** |

### Rejection Sampling（拒绝采样）

LLaMA 的核心创新之一。流程如下：

1. 准备 50–100 万条提示词；
2. 让当前最强模型对每条提示词生成 **20–30 个**不同回答；
3. 用 RM 对每个回答打分；
4. **Rejection Sampling**：取每组中分数最高的前 3 名，淘汰其余；
5. 每条提示词保留 3 个高质量回答，构成 SFT 训练数据（一问三答，展开为 3 条问答对）；
6. 用这批 AI 筛选的数据进行 SFT 训练，得到更强的模型。

这实质上是让 **AI（RM 模型）完成数据标注工作**，取代了大量人工标注，显著降低了数据获取成本。

### DPO（Direct Preference Optimization，直接偏好优化）

DPO 是 LLaMA 用于替代 PPO 强化学习的对齐方法，其思路更接近 SFT 的扩展，而非传统强化学习。

**核心思想：**

SFT 阶段只告诉模型什么是"好的回答"（最大化好回答的段落联合概率），而 DPO 同时利用偏好数据中的**好回答 A** 和**差回答 B**，定义双向损失函数：

$$\mathcal{L}_{\text{DPO}} = -\mathbb{E}\left[\log\sigma\left(\beta \log\frac{\pi_\theta(y_w|x)}{\pi_{\text{ref}}(y_w|x)} - \beta \log\frac{\pi_\theta(y_l|x)}{\pi_{\text{ref}}(y_l|x)}\right)\right]$$

直觉上即：好回答的段落联合概率越高越好，差回答的段落联合概率越低越好，最大化二者之差。

**DPO 与 PPO 的区别：**

- PPO（强化学习）：需要先训练 RM，再用 RM 打分驱动大模型参数更新；整个流程更复杂；
- DPO：直接用偏好数据（好回答 vs. 差回答）优化大模型，绕过 RM 的中间打分环节，更简洁。

注意：学术上，DPO 不完全等同于强化学习；其优化逻辑与 SFT 更为相似。但工程实践和面试场合中，DPO 和 PPO 常被并列为"对齐阶段的两种方法"。

### LLaMA 后训练的完整循环

LLaMA 的后训练为迭代循环，每轮步骤如下：

1. 收集偏好数据（人类对两个回答进行对比标注）；
2. 训练 RM（奖励模型）；
3. RM 执行 Rejection Sampling，筛选出高质量 SFT 数据；
4. 结合领域专项数据，进行 SFT 训练，得到新的当前最强模型；
5. 用最新偏好数据对新模型进行 DPO 优化；
6. 新模型成为下一轮的"当前最强模型"，重复上述流程。

LLaMA 3 系列共进行了约 **6 轮**这样的循环，每轮的数据质量递进（从好的回答中再选更好的），教练员（RM）也随之不断升级。

## 附：模型量化简介

**量化（Quantization）**是一种压缩模型以降低部署成本的技术，与训练范式无关，但常在部署场景下讨论。

正常情况下，每个模型参数用 **FP32（32位浮点数，4 字节）** 存储。量化将其压缩为更小的格式：

| 精度 | 字节/参数 | 存储压缩比 |
|------|---------|---------|
| FP32 | 4 字节 | 1× |
| FP16 / BF16（半精度） | 2 字节 | 2× |
| INT8（8位整数） | 1 字节 | 4× |

以 DeepSeek 671B 为例：$671 \times 10^9 \times 4 \approx 2684\text{ GB}$（FP32）。部署时至少需要约 2700 GB 显存；若量化至 INT8，则约需 675 GB。训练时所需显存通常为部署的 4–8 倍，即 10000–20000 GB 以上。

量化的代价是**精度损失**：参数从精确浮点数被近似为整数，模型的推理质量会有所下降，压缩比越高损失越明显。INT8 量化已开始产生可感知的质量下降，FP16 损失则相对较小。

## 附：知识蒸馏简介

**知识蒸馏（Knowledge Distillation）**是用大模型（教师模型）生成的输出数据，来训练小模型（学生模型）的技术。

典型场景：用 DeepSeek 671B 对 100 万条提示词生成回答，再用这些问答对对 DeepSeek 32B 进行 SFT 微调，可显著提升 32B 模型的能力。

**核心局限：** 学生模型的上限受其自身参数量约束——671B 能做的所有任务，32B 绝无可能完全习得。蒸馏更适合用于：

- 模型能力冷启动（如早期 Doubao 从 GPT 蒸馏数据开始训练）；
- 让小模型学会大模型擅长的某一**子集**任务；

而非期望小模型完全复制大模型的全部能力。

## 重要术语汇总

| 术语 | 全称 / 含义 |
|------|-----------|
| Pre-training | 预训练，使用海量无标注文本训练的第一阶段 |
| SFT | Supervised Fine-Tuning，有监督微调，使用问答对数据 |
| Reward Model (RM) | 奖励模型，用于给大语言模型输出打分的分支模型 |
| Preference Data | 偏好数据，人类对多个回答进行排序产生的标注数据 |
| RLHF | Reinforcement Learning from Human Feedback，基于人类反馈的强化学习 |
| PPO | Proximal Policy Optimization，GPT 使用的强化学习算法 |
| DPO | Direct Preference Optimization，直接偏好优化，LLaMA 使用的对齐方法 |
| Rejection Sampling | 拒绝采样，RM 从多个生成回答中筛选高质量数据的过程 |
| Alignment | 对齐，使模型输出与人类期望、价值观保持一致 |
| Instruction Following | 指令遵循，模型理解并执行用户指令的能力 |
| Joint Probability | 段落联合概率，预训练和 SFT 阶段的核心训练目标 |
| Epoch | 训练轮次，模型将所有训练数据完整学习一遍 |
| Batch | 批次，一次参数更新使用的训练数据子集 |
| Token | 模型处理文本的基本单位，约 0.7–0.8 个汉字 |
| Quantization | 量化，压缩模型参数精度以降低存储和计算开销 |
| Knowledge Distillation | 知识蒸馏，用大模型输出数据训练小模型的技术 |
| Exploration | 探索，强化学习中尝试非最优选择以发现更好路径 |
| Exploitation | 遵循，强化学习中依赖已知最优策略的行为 |

## 总结

大语言模型的训练范式经过多年演进，已形成以下稳定框架：

**预训练 → SFT → 奖励模型训练 → 强化学习对齐**

以 GPT 为代表的闭源路线与以 LLaMA 为代表的开源路线，整体框架高度一致，差异主要体现在后训练阶段的具体方法选择上：GPT 使用 PPO 强化学习；LLaMA 使用 Rejection Sampling 筛选数据 + DPO 直接优化，并以约 6 轮迭代循环完成对齐。
