## 核心原理

### 神经网络与大语言模型基础

神经网络通过大量训练数据和参数调整，目标是降低误差（损失函数），不断优化参数使模型输出更准确。传统单一神经网络难以高效处理复杂语言任务，因此大语言模型设计为多阶段、多模块协同完成特定功能。

### Transformer模型框架

Transformer是一个**模型框架**，将大语言模型的计算过程划分为多个阶段，每个阶段承担不同计算任务。每次生成一个token时，模型内部经历完整的多阶段计算流程。输入部分存储用户输入的问题或提示词，输出部分逐字生成回答。

### Tokenization（词元化）

文本首先被拆分成**token**，即子词单元。Token可以是完整单词，也可以拆分为多个子词，目的是减少token总数，降低计算复杂度。不同模型的token拆分策略不同，影响计算效率和生成速度。

| 模型       | 语言       | 词数（约） | Token数（约） | 备注                         |
|------------|------------|------------|--------------|------------------------------|
| GPT-3.5    | 英文       | 750        | 1000         | 英文单词与token比例约1:1.3   |
| GPT-3.5    | 中文       | 650        | 1300         | 汉字平均对应约2个token       |
| GLM        | 英文/中文  | —          | 40万+        | 中文token数较英文更接近     |

Tokenization是固定映射规则，不涉及神经网络参数。

### Embedding（嵌入）

每个token被映射为一个高维数学向量（如GPT-3为12288维），这些向量是模型参数，训练过程中不断调整。Embedding向量密集且能体现词语间的语义相似性和逻辑关系，远优于传统的One-hot编码。

示例：  
“国王”向量与“皇帝”、“君王”相似；“狗”与“猫”相似；通过向量加减法表达语义关系，如“国王” - “男人” + “女人” ≈ “女王”。

### 位置编码（Positional Encoding）

Transformer一次性并行处理所有token，缺乏序列顺序感知能力。位置编码为每个token提供位置信息的向量，与token embedding相加，形成包含语义和位置信息的综合向量，使模型理解词序对语义的影响。

位置编码最初为固定数学公式，后演变为可训练参数，动态学习词序信息。

## 技术细节

### 编码器（Encoder）与解码器（Decoder）

Transformer核心模块包括编码器和解码器：

- **编码器**负责将输入文本转为高维向量，捕捉语义信息，适合文本理解和分析。编码器输入输出的token embedding数量保持不变，经过多轮信息聚合，向量信息丰富。
- **解码器**基于编码器输出和已生成文本，逐字生成新文本。解码器输入包括编码器输出和已生成token的embedding，随着生成文本长度增加，输入项量数量动态增长。解码器采用前向限制（Mask机制），保证自回归生成的因果性。

三种架构类型对比：

| 架构类型       | 组成模块           | 功能定位         | 典型代表           | 说明                                                         |
| -------------- | ------------------ | ---------------- | ------------------ | ------------------------------------------------------------ |
| Encoder-Decoder | 编码器 + 解码器    | 分析 + 生成      | 经典Transformer   | 输入先编码，编码结果传递给解码器，解码器生成输出。           |
| Decoder-Only   | 仅解码器           | 生成（兼具理解） | GPT系列（GPT-1起） | 解码器既理解输入（提示词），又生成输出，无需编码器。         |
| Encoder-Only   | 仅编码器           | 分析             | BERT               | 只做文本理解和表示，不生成文本。                             |

### 信息聚合机制

信息聚合是Transformer理解文本的核心过程。每个token向量作为中心，通过计算与其他token向量的相似度，聚合相关信息，丰富自身语义表示。多轮信息聚合使token向量逐渐融合上下文，形成短语级别甚至全文级别的语义理解。

编码器允许token访问全文所有token，解码器仅能访问当前token及之前token，保证生成的自回归性质。

### 训练流程与参数自省

训练任务为基于前n个token预测第n+1个token，输出token概率分布。模型参数（如GPT-3约1750亿）通过误差梯度不断调整（自省），优化整体误差。训练采用分批次迭代，预训练通常需数十天。

训练和推理阶段计算流程基本一致，推理阶段参数固定，模型基于已有参数生成文本。

### 生成过程详解

- **Linear层**将解码器输出的高维向量映射到token表大小的向量，用于计算与每个token的相似度。
- **Softmax层**将相似度转换为概率分布，生成概率最高的token。
- **自回归机制**逐步生成文本，每生成一个token加入输入序列，模型重复计算。

### 位置编码与上下文长度

位置编码数量与模型支持的上下文长度直接相关，如GPT-3支持2048个token上下文，对应2048个位置编码向量。长文本处理带来存储和计算挑战，研究者提出旋转位置编码等技术优化。

### 向量加法与维度要求

Embedding向量与位置编码向量维度必须相同（如均为12288维），才能逐元素相加形成新的向量，融合语义与位置信息。

## 案例分析

### 以“中秋节吃什么”为例的生成流程

用户输入“中秋节吃什么”，经过Tokenization拆分为多个token，映射为高维embedding向量。位置编码向量与token embedding相加，形成带位置信息的输入向量。编码器对输入进行多轮信息聚合，提取语义特征。解码器基于编码器输出和已生成文本，逐字生成回答，如“月饼”、“柚子”等，直到生成结束token。

### GPT-3模型参数规模与训练

GPT-3拥有约1750亿参数，Token表大小约50257个token，最大上下文长度2048 token，向量维度12288维，信息聚合层数约96层。训练数据规模巨大，训练时间长，参数规模保证了强大的拟合能力和复杂语义捕捉能力。

### 多模态模型的向量统一

多模态模型将不同类型数据（文字、图片、音频、视频）统一映射为高维向量，模型基于这些向量进行理解和生成，实现跨模态融合。

## 总结

Transformer通过多阶段计算和信息聚合机制，实现对语言的深度理解和生成。Tokenization和Embedding将文本转换为数学向量，位置编码补充序列顺序信息。编码器与解码器分别承担理解和生成任务，协同完成复杂应用。多轮信息聚合使模型捕捉上下文和复杂语义关系。大规模参数提升模型拟合能力，但需平衡参数与数据规模。未来模型将继续优化token拆分、向量表达和位置编码技术，提升效率和效果。

## 术语表

| 术语               | 说明                                                         |
|--------------------|--------------------------------------------------------------|
| Token              | 文本的最小单位，可能是字、词或子词                           |
| Embedding（项量）  | Token对应的高维向量表示，含语义和位置信息                   |
| 位置编码（Positional Encoding） | 为token添加位置信息的向量，保证序列顺序感知               |
| 信息聚合（Information Aggregation） | 多轮次计算融合token信息，提取语义特征                       |
| Mask机制           | 解码器训练时限制token只能访问前文，防止“偷看”未来信息       |
| Linear层           | 将高维向量映射到token空间，计算相似度                         |
| Softmax层          | 将相似度转换为概率分布，概率和为1                             |
| 自回归（Autoregressive） | 逐步生成文本，每次基于已有文本预测下一个token               |
| Decoder-Only架构   | 仅使用解码器模块，兼具理解与生成能力的Transformer变体        |
| Encoder-Only架构   | 仅使用编码器模块，专注于文本理解的Transformer变体             |
| 参数（Parameters）  | 模型中可调节的权重，决定模型的表达能力                       |
| RAG                | 基于检索增强生成的模型，结合知识库进行问答                   |
| ASR                | 自动语音识别模型，将音频转换为文字                           |
| 多模态（Multimodal） | 同时处理多种数据类型（文本、图像、音频等）的模型               |
