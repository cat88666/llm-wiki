# 提示工程（Prompt Engineering）

## 技术本质

- 应用层技术的核心，往往是在拼凑出一条 **合适的 Prompt**。
- Prompt 是与 **大语言模型（LLM）沟通的主要方式**。

## Prompt 的常见组成

- **身份设定**
- **背景设定**
- **参考资料**
- **样例（示例）**
- **指令**
- **限制条件**

## 学习模式

| 模式 | 说明 |
|------|------|
| **Zero-Shot** | 零样本：不提供示例，直接提问。 |
| **One-Shot / Few-Shot** | 提供一个或多个样例，引导模型按期望格式或风格输出。 |

## 局限性

- 提示词存在 **字数 / 长度限制**。
- 内容过多时，可能导致 **模型性能明显下降**（注意力分散、关键信息被稀释等）。
