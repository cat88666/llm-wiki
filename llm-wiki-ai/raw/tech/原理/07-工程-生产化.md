## 一、为什么生产化是独立挑战

Demo 能跑 ≠ 可以上线。LLM 应用的生产化挑战：

- **不确定性**：同样的输入可能产生不同输出
- **成本不可控**：Token 用量随用户行为波动
- **幻觉难以发现**：错误输出不会报错，只会误导用户
- **延迟不稳定**：外部 API 依赖，网络和模型负载影响延迟
- **安全边界模糊**：用户可能尝试绕过 Prompt 约束

**一句话**：Prompt 决定行为边界，RAG 决定知识来源，Tool 决定行动能力，**Observability 决定系统能否真正上线**。

## 二、评估体系

### 2.1 RAG 评估（Ragas 框架）

| 指标 | 评测内容 | 计算方式 |
|------|----------|----------|
| Faithfulness（忠实度） | 答案是否完全基于上下文 | 答案中可被上下文支撑的陈述比例 |
| Answer Relevancy | 答案是否回答了问题 | 生成答案反推问题与原问题的相似度 |
| Context Recall | 相关信息是否被检索到 | 标准答案中能在上下文中找到的比例 |
| Context Precision | 检索结果是否精准 | 上下文中真正有用的块的比例 |

**Faithfulness 是最重要的指标**：低 Faithfulness 意味着答案不基于检索结果，即幻觉。

### 2.2 LLM-as-Judge（自动化评估）

```java
String evalPrompt = """
    请评估以下 AI 回答的质量，返回 JSON：
    
    用户问题：%s
    参考资料：%s
    AI 回答：%s
    
    评估标准：
    - faithfulness: 回答是否忠于参考资料（0-5）
    - relevance: 回答是否切题（0-5）
    - completeness: 是否回答了问题的所有方面（0-5）
    - hallucination: 是否包含参考资料中没有的事实（true/false）
    
    返回格式：{"faithfulness": X, "relevance": X, "completeness": X, "hallucination": true/false, "reason": "..."}
    """.formatted(question, context, answer);
```

### 2.3 建立评估集

**黄金评估集**：
- 覆盖业务高频场景（至少 100 条问答对）
- 包含边界情况（信息不足、跨文档推理、歧义问题）
- 有人工标注的标准答案和判分标准

**每次改动后跑回归测试**：修改 Prompt、更换模型、调整 chunk size 后，必须在评估集上验证不退化。

## 三、可观测性（Observability）

### 3.1 必须追踪的指标

```
每次 LLM 调用记录：
├── trace_id（链路追踪 ID）
├── user_id（用户标识）
├── session_id（会话 ID）
├── model_name + model_version
├── prompt_version（Prompt 版本号）
├── input_tokens + output_tokens → 成本计算
├── latency_ttft（首 token 延迟）
├── latency_total（总延迟）
├── retrieved_chunks（RAG 召回的 chunk 数）
├── retrieved_scores（相似度分数列表）
├── user_feedback（点赞/踩，事后收集）
└── error（如有）
```

### 3.2 LangFuse 集成

```java
// 依赖
// implementation 'com.langfuse:langfuse-java:1.x.x'

@Service
public class TracedRagService {

    private final Langfuse langfuse = new Langfuse(apiKey, secretKey, host);

    public String query(String userId, String question) {
        Trace trace = langfuse.trace(TraceCreateParams.builder()
            .name("rag-query")
            .userId(userId)
            .input(question)
            .metadata(Map.of("promptVersion", PROMPT_VERSION))
            .build());

        // 记录检索
        Span retrievalSpan = trace.span(SpanCreateParams.builder()
            .name("retrieval")
            .input(question)
            .build());
        
        List<Document> docs = vectorStore.similaritySearch(question);
        retrievalSpan.end(SpanUpdateParams.builder()
            .output(docs.size() + " chunks retrieved")
            .build());

        // 记录 LLM 调用
        Generation generation = trace.generation(GenerationCreateParams.builder()
            .name("llm-call")
            .model("claude-sonnet-4-6")
            .input(buildPrompt(question, docs))
            .build());

        String answer = llm.generate(buildPrompt(question, docs));
        
        generation.end(GenerationUpdateParams.builder()
            .output(answer)
            .usage(UsageParams.builder()
                .inputTokens(tokenizer.count(question))
                .outputTokens(tokenizer.count(answer))
                .build())
            .build());

        trace.update(TraceUpdateParams.builder().output(answer).build());
        return answer;
    }
}
```

### 3.3 关键监控告警

```
告警规则：
- Faithfulness 平均分 < 3.5（最近1小时）→ 立即告警
- 幻觉率 > 5%（最近1小时）→ 立即告警
- 平均延迟 > 10s → 告警
- Error 率 > 1% → 告警
- 单用户 Token 消耗异常（>100K/小时）→ 告警
```

## 四、成本控制

### 4.1 模型分级路由

```
简单问题（意图分类、FAQ 匹配）
  → claude-haiku-4 / gpt-4o-mini
  → 成本：约 $0.0001/1K tokens

中等问题（RAG 问答）
  → claude-sonnet-4-6
  → 成本：约 $0.003/1K tokens

复杂推理（代码生成、深度分析）
  → claude-opus-4-6 / gpt-4o
  → 成本：约 $0.015/1K tokens
```

```java
public String route(String question) {
    if (isSimpleIntent(question)) return "claude-haiku-4-5-20251001";
    if (requiresDeepReasoning(question)) return "claude-opus-4-6";
    return "claude-sonnet-4-6";  // 默认
}
```

### 4.2 语义缓存

```java
@Service
public class SemanticCache {
    
    private static final double SIMILARITY_THRESHOLD = 0.92;
    private final EmbeddingModel embeddingModel;
    private final RedisTemplate<String, CacheEntry> redis;

    public Optional<String> get(String question) {
        float[] queryVector = embeddingModel.embed(question).content().vector();
        
        // 从 Redis 取最近缓存的问题向量
        List<CacheEntry> candidates = getRecentCacheEntries();
        
        return candidates.stream()
            .filter(entry -> cosineSimilarity(queryVector, entry.vector()) > SIMILARITY_THRESHOLD)
            .max(Comparator.comparingDouble(e -> cosineSimilarity(queryVector, e.vector())))
            .map(CacheEntry::answer);
    }

    public void put(String question, String answer) {
        float[] vector = embeddingModel.embed(question).content().vector();
        CacheEntry entry = new CacheEntry(question, answer, vector, Instant.now());
        redis.opsForValue().set(
            "semantic_cache:" + UUID.randomUUID(),
            entry,
            Duration.ofHours(24)
        );
    }
}
```

**缓存适用场景**：
- FAQ 类重复问题（命中率高）
- 报表类固定查询
- 不适用：个性化、实时性要求高的场景

### 4.3 Prompt 压缩

```
对话历史压缩：
  超过 N 轮后，用轻量模型（haiku）压缩历史为摘要
  压缩比例：10 轮 → 200 tokens 摘要

文档上下文压缩：
  检索到 10 个 chunk，先用轻量模型提取与问题相关的段落
  再把精选片段拼入主模型 Prompt

批处理优化：
  非实时任务（报告生成、数据标注）使用 Batch API
  成本降低 50%，延迟从秒级变为分钟级
```

### 4.4 Token 成本监控

```java
// 每次 LLM 调用后记录成本
double cost = (inputTokens * inputPricePerToken) + (outputTokens * outputPricePerToken);
metricsService.recordCost(userId, sessionId, cost);

// 每日成本报告
SELECT user_id, SUM(cost) as daily_cost, COUNT(*) as queries
FROM llm_calls
WHERE created_at > NOW() - INTERVAL '1 DAY'
GROUP BY user_id
ORDER BY daily_cost DESC;
```

## 五、可靠性工程

### 5.1 超时与重试

```java
@Retry(maxAttempts = 3, backoff = @Backoff(delay = 1000, multiplier = 2))
@TimeLimiter(name = "llmCall", fallbackMethod = "fallback")
public String callLLM(String prompt) {
    return llmClient.generate(prompt);
}

// 降级策略
public String fallback(String prompt, Exception e) {
    log.warn("LLM call failed, using fallback", e);
    return "抱歉，AI 服务暂时繁忙，请稍后重试。如有紧急需求，请联系人工客服。";
}
```

### 5.2 模型降级

```java
public String generateWithFallback(String prompt) {
    List<String> modelPriority = List.of(
        "claude-sonnet-4-6",      // 主模型
        "claude-haiku-4-5-20251001", // 降级1：成本更低
        "deepseek-chat"            // 降级2：备用
    );
    
    for (String model : modelPriority) {
        try {
            return llmClient.generate(model, prompt);
        } catch (ServiceUnavailableException e) {
            log.warn("Model {} unavailable, trying next", model);
        }
    }
    throw new AllModelsUnavailableException();
}
```

## 六、安全防护（Guardrails）

### 6.1 输入安全

```java
public void validateInput(String userInput) {
    // 长度限制
    if (userInput.length() > 5000) {
        throw new InputTooLongException("输入超过5000字符");
    }
    
    // 注入检测
    List<String> injectionPatterns = List.of(
        "ignore previous instructions",
        "ignore all instructions",
        "你现在是",
        "system:",
        "SYSTEM:",
        "\\[INST\\]"
    );
    for (String pattern : injectionPatterns) {
        if (userInput.toLowerCase().contains(pattern.toLowerCase())) {
            auditLog.warn("Potential prompt injection: user={}, input={}", userId, userInput);
            throw new SecurityException("检测到异常输入");
        }
    }
}
```

### 6.2 输出安全

```java
public String filterOutput(String llmOutput) {
    // 手机号脱敏
    llmOutput = llmOutput.replaceAll("1[3-9]\\d{9}", "1****${group1}${group2}");
    // 身份证脱敏
    llmOutput = llmOutput.replaceAll("\\d{17}[\\dX]", "***");
    // 银行卡号脱敏
    llmOutput = llmOutput.replaceAll("\\d{16,19}", "****");
    return llmOutput;
}
```
