## 一、线性模型

**Q: 为什么逻辑回归是分类模型而不是回归模型？**

逻辑回归用 sigmoid 函数将线性输出映射到 (0,1)，输出的是"属于正类的概率"，优化目标是最大化对数似然（等价于最小化交叉熵）。名字里有"回归"是历史原因。

---

**Q: 逻辑回归为什么不用 MSE 损失？**

分类任务用 MSE 损失是非凸函数，有多个局部最小值，优化困难。交叉熵损失是凸函数，有唯一全局最小值，梯度下降更稳定。

---

**Q: L1 和 L2 正则化的区别？什么场景用哪个？**

| | L1（Lasso） | L2（Ridge） |
|---|---|---|
| 惩罚项 | 权重绝对值之和 | 权重平方和 |
| 解的特点 | 稀疏（部分权重为0） | 权重均匀缩小 |
| 几何直觉 | 约束域为菱形，梯度方向易切顶点 | 约束域为球，梯度方向连续 |
| 适用 | 特征选择、高维稀疏 | 特征共线性、稳定训练 |

面试一句话：**L1 选特征，L2 防过拟合**。

---

**Q: LR 和 SVM 的区别？**

核心差异：
- LR 是概率模型，输出概率；SVM 是间隔模型，输出决策值
- LR 用交叉熵损失，所有样本都参与优化；SVM 用 hinge loss，只有支持向量决定超平面
- LR 可直接得到概率输出；SVM 需额外 Platt scaling 校准概率

工程选择：需要概率输出/可解释性/大规模线性分类 → LR；需要最大间隔分类 → SVM。

## 二、SVM

**Q: SVM 为什么最大化间隔能提高泛化能力？**

最大化间隔等价于最小化 $\|w\|^2$，这是一种正则化效果（限制了参数空间）。更大的间隔意味着对训练数据中的噪声更鲁棒——轻微扰动的样本不容易越过边界。

---

**Q: 什么是支持向量？为什么叫"支持"？**

支持向量是距离分类超平面最近的样本点，它们"支撑"起分类边界。关键特性：只有这些点决定超平面位置，删除其他所有样本，模型不变。这体现了 SVM 只关注最困难的边界样本。

---

**Q: 核函数是什么，为什么有效？**

核函数 $K(x_i, x_j) = \phi(x_i)^T \phi(x_j)$ 隐式计算高维空间中的内积，无需显式映射。

为什么有效：在原始空间线性不可分的数据，在足够高维的空间可能变得线性可分。核技巧用 O(n) 的计算完成了 O(d_high) 的变换效果。

常见核：线性核（无映射）、多项式核、RBF 核（默认最常用）。

## 三、树模型与集成学习

**Q: 决策树为什么容易过拟合？如何解决？**

决策树可以无限细分直到每个叶节点只有一个样本，本质是没有约束的非参数模型。

解法：
- 预剪枝：限制树深、最小叶节点样本数
- 后剪枝：先建完整树再剪
- 集成方法：用 Bagging（随机森林）或 Boosting（GBDT）

---

**Q: 随机森林为什么比单棵决策树泛化更好？**

两个随机化（Bootstrap 采样 + 特征随机选择）确保多棵树之间弱相关。方差可加性：n 个不相关模型的平均，方差降低为原来的 1/n。即使每棵树高方差（过拟合），平均后方差大幅下降，偏差基本不变。

---

**Q: GBDT 的核心思想是什么？**

每一轮训练一棵新树去拟合当前模型的负梯度（残差），通过加法模型逐步纠正错误：$F_t(x) = F_{t-1}(x) + \eta \cdot h_t(x)$，其中 $h_t$ 是拟合残差的新树，$\eta$ 是学习率（缩减策略）。

---

**Q: XGBoost 相比传统 GBDT 有哪些改进？**

算法改进：
1. **二阶泰勒展开**：利用梯度和 Hessian（二阶导），对损失函数近似更精确，收敛更快
2. **正则化**：对树结构加入 L1+L2 正则，防过拟合
3. **缺失值自动处理**：训练时学习每个节点的最优缺失值方向

工程改进：
4. **列采样**：类似随机森林，每次分裂随机选特征，防过拟合且加速
5. **近似分位数算法**：不枚举所有分裂点，大数据集更高效
6. **并行化**：特征预排序 + 并行寻找最优分裂点

---

**Q: XGBoost 为什么用泰勒展开？（高频考点）**

泰勒展开将任意损失函数在当前预测值处近似为二次函数：$L(y, F+h) \approx L(y, F) + g \cdot h + \frac{1}{2}h \cdot h \cdot h$（其中 g 是一阶导，h 是二阶导）。

好处：
- 对任意可微损失函数都适用（不依赖具体损失函数形式）
- 二次函数有解析最优解，直接求出最优叶节点权重
- 二阶信息让步长更合理，收敛更快

## 四、优化方法

**Q: 梯度下降是最快的下降方向吗？**

只是当前点**一阶意义下**的最陡下降方向，不是全局最快收敛路径。梯度方向只保证在极小邻域内最快，离当前点远了就不再保证。牛顿法用二阶信息可以找到更优方向，但计算 Hessian 矩阵代价高。

---

**Q: 为什么深层网络容易梯度消失/爆炸？如何解决？**

原因：反向传播是链式法则连乘，若每层梯度 < 1，连乘后趋零（消失）；> 1 则趋于无穷（爆炸）。

解法：
- 消失：ReLU 激活（正区间梯度为1）、残差连接（梯度直接传递）、LSTM 门控机制
- 爆炸：梯度裁剪（$g = g \cdot \frac{c}{\|g\|}$ 当 $\|g\| > c$）
- 两者：LayerNorm/BatchNorm、合理初始化（Xavier/He）

## 五、模型评估

**Q: Accuracy、Precision、Recall、F1 分别什么时候用？**

- **Accuracy**：类别均衡时可用，不平衡时无意义
- **Precision**：关注"预测为正的样本有多少真的是正"，适合误报代价高的场景（垃圾邮件过滤）
- **Recall**：关注"所有正样本被找到了多少"，适合漏报代价高的场景（疾病筛查）
- **F1**：Precision 和 Recall 的调和平均，两者都重要时用

---

**Q: AUC 的意义是什么？为什么它比 Accuracy 更好？**

AUC 的概率意义：随机取一个正样本和一个负样本，模型给正样本打分高于负样本的概率。

好处：
- 不依赖分类阈值，评估整体排序能力
- 对类别不平衡不敏感（不受负样本多少影响）
- AUC = 0.5 表示随机，AUC = 1.0 表示完美

---

**Q: 类别极不平衡（如 1:1000）怎么处理？**

```
数据层：
  - 过采样少数类（SMOTE 合成新样本）
  - 欠采样多数类
  
训练层：
  - class_weight='balanced'（自动根据频率加权）
  - Focal Loss（对困难样本加大权重）
  
评估层：
  - 换指标：用 F1、AUC、PR-AUC，不用 Accuracy
  - 调整决策阈值（不一定用 0.5）
```

## 六、特征工程

**Q: 为什么说特征工程比换模型更重要？**

数据决定上限，算法决定能逼近上限多少。好的特征让最简单的逻辑回归都能打败复杂模型；差的特征让再强的模型也无法弥补信息缺失。特征工程是杠杆最大的投入。

---

**Q: 哪些模型需要做特征归一化？哪些不需要？**

**需要**：LR、SVM（基于距离/梯度尺度）、KNN、KMeans、神经网络

**不需要**：决策树、随机森林、GBDT/XGBoost（按阈值分裂，不依赖特征数值尺度）

---

**Q: 如何处理缺失值？**

| 方法 | 适用场景 |
|------|----------|
| 均值/中位数填充 | 连续值，缺失少 |
| 众数填充 | 类别值 |
| 模型预测填充 | 缺失有规律 |
| 单独标记（is_missing 特征）| 缺失本身可能有信息 |
| 删除（行/列）| 缺失比例极高，无法补救 |

树模型（XGBoost）可直接处理缺失值，无需填充。

## 七、完整 ML 项目

**Q: 如何设计一个完整的机器学习项目？（开放题）**

```
1. 问题定义：业务目标 → 机器学习任务类型 → 评估指标
2. 数据探查：分布 / 缺失 / 异常 / 标签质量
3. 特征工程：清洗 → 构造 → 筛选 → 归一化
4. 建模：基线（简单模型）→ 复杂模型 → 交叉验证调参
5. 误差分析：错误样本分析，是数据问题还是特征问题
6. 模型融合：Stacking / Blending
7. 上线监控：数据漂移 / 性能衰减
```

**回答技巧**：不要只背流程，要补一句"真正的项目是持续迭代的闭环，不是一次性流程"。
