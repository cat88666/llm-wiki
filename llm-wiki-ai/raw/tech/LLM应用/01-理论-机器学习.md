# 机器学习原理

## 一、学习体系总览

机器学习的本质：**从数据中归纳规律，用规律对未见样本做预测或决策**。

```
任务类型
├── 监督学习（有标签）
│   ├── 分类：预测离散类别
│   └── 回归：预测连续值
├── 无监督学习（无标签）
│   ├── 聚类：找数据内在分组
│   └── 降维：压缩表示
└── 强化学习：智能体与环境交互
```

**判别式 vs 生成式模型**：

| | 判别式 | 生成式 |
|---|---|---|
| 学习目标 | 直接学 P(y\|x) 或决策边界 | 学联合分布 P(x,y) 或 P(x\|y) |
| 代表 | LR、SVM、决策树、CRF | 朴素贝叶斯、GMM、HMM、LDA |
| 特点 | 通常分类效果更好 | 可以生成新样本 |

## 二、线性模型

### 2.1 线性回归

**核心思想**：假设输出是输入特征的线性组合，通过最小化均方误差学习参数。

$$\hat{y} = w^T x + b, \quad L = \frac{1}{n}\sum_i(y_i - \hat{y}_i)^2$$

**优点**：简单、可解释、作为基线模型有价值。
**局限**：只能拟合线性关系，对异常值敏感。

---

### 2.2 逻辑回归

**为什么不用线性回归做分类**：线性回归输出实数域，分类需要概率输出。Sigmoid 将线性输出压缩到 (0,1)：

$$P(y=1|x) = \sigma(w^Tx) = \frac{1}{1+e^{-w^Tx}}$$

**损失函数**：交叉熵（对数似然的负值），而非 MSE。MSE 在概率模型上是非凸的，优化困难。

**多分类扩展**：Softmax 回归，输出归一化概率向量。

---

### 2.3 正则化：控制模型复杂度

| | L1（Lasso） | L2（Ridge） |
|---|---|---|
| 惩罚项 | $\lambda\sum\|w_i\|$ | $\lambda\sum w_i^2$ |
| 几何直觉 | 约束域是菱形，容易压到顶点（稀疏解） | 约束域是球，权重均匀缩小 |
| 概率解释 | 拉普拉斯先验 | 高斯先验 |
| 适用场景 | 特征选择、高维稀疏 | 特征共线性、稳定训练 |

**Elastic Net** = L1 + L2，兼顾稀疏性和稳定性。

## 三、支持向量机（SVM）

### 3.1 核心思想：最大化间隔

SVM 不只是"把样本分开"，而是找到**间隔最大**的超平面。间隔最大等价于 $\min\|w\|^2$，天然带有正则化效果。

**支持向量**：仅边界上的少数样本决定超平面。模型由支持向量完全确定，删除其他样本不影响结果。

三种形式：
- **硬间隔**：严格线性可分，不容忍错误
- **软间隔**：允许少量违例，实际场景更常用
- **非线性 SVM**：通过核函数映射到高维，再做线性划分

### 3.2 核技巧

核函数在不显式计算高维映射的情况下，完成高维内积计算：$K(x_i, x_j) = \phi(x_i)^T\phi(x_j)$。

常见核：线性核、多项式核、RBF（高斯核）。RBF 是最常用的默认选择，能处理任意维度的非线性边界。

## 四、树模型与集成学习

### 4.1 决策树

**划分指标**（选哪个特征分裂）：

| 用途 | 指标 | 含义 |
|---|---|---|
| 分类 | 信息增益 | 划分后熵减少了多少 |
| 分类 | 基尼指数 | 随机猜一个样本分错的概率 |
| 回归 | 均方差下降 | 划分后方差减少了多少 |

**天然缺陷**：对数据扰动敏感，容易过拟合——这是集成学习的动机。

---

### 4.2 随机森林（Bagging）

**两个随机化**：
1. Bootstrap 采样：每棵树用不同的有放回样本
2. 特征随机：每次分裂只从随机子集特征中选最优

**为什么有效**：多棵弱相关的树取平均，偏差不变但方差大幅降低。
**特征重要性**：Gini 下降（默认）/ OOB 误差变化 / 打乱后性能下降。

---

### 4.3 梯度提升（GBDT）

**核心思想**：每一轮拟合前一轮模型的**负梯度**（残差），逐步纠正错误。

```
初始模型 f₀(x) = 常数
第 t 轮：
  1. 计算负梯度 r_i = -∂L/∂f(x_i)（残差）
  2. 用新决策树 h_t(x) 拟合 r_i
  3. f_t(x) = f_{t-1}(x) + η·h_t(x)
```

**超参数**：学习率 η（越小越稳）、树数量（越多越好但有上限）、树深（控制过拟合）。

---

### 4.4 XGBoost：工程化的 GBDT

**算法改进**：
- 用**二阶泰勒展开**近似损失函数，利用一阶+二阶导数，收敛更快
- 加入 L1/L2 正则化到树结构评分，泛化更好
- 自动处理缺失值（学习最优分裂方向）

**工程改进**：
- 列采样（Column Subsampling）：类似随机森林的特征随机，防过拟合
- 近似分位数算法：不枚举所有分裂点，效率更高
- 并行特征分裂：GPU 支持，训练更快

**XGBoost vs GBDT 核心差异**：

| | GBDT | XGBoost |
|---|---|---|
| 梯度信息 | 一阶 | 一阶 + 二阶 |
| 正则化 | 无 | L1 + L2 |
| 并行 | 有限 | 成熟 |
| 缺失值 | 需预处理 | 自动处理 |

### 4.5 LightGBM

LightGBM 在 XGBoost 基础上进一步优化速度：
- **直方图算法**：将连续特征分桶，减少分裂候选点
- **Leaf-wise 生长**：不按层生长而是按最大增益叶子生长，同等叶子数效果更好
- **GOSS（梯度单边采样）**：保留大梯度样本，随机抽样小梯度样本

## 五、贝叶斯方法

### 5.1 朴素贝叶斯

**核心假设**：给定类别，各特征条件独立（朴素假设）。

$$P(y|x) \propto P(y) \prod_i P(x_i|y)$$

**为什么叫"朴素"**：条件独立假设现实中几乎不成立，但在文本分类等高维稀疏场景中依然有效——词袋模型下，统计足够时独立假设的误差影响有限。

**EM 算法**：用于含隐变量的参数估计，E 步估计隐变量分布，M 步最大化期望似然，循环直到收敛。适用场景：GMM、HMM、主题模型 LDA。

## 六、聚类

### 6.1 KMeans

```
1. 随机初始化 k 个中心
2. 每个样本分配给最近中心
3. 重新计算每个簇的中心（均值）
4. 重复 2-3 直到收敛
```

**局限**：
- 对初始化敏感（KMeans++ 改进：初始中心尽量分散）
- 对异常值敏感（均值被拉偏）
- 需要预设 k
- 只适合凸形簇

**为什么 KMeans 用欧氏距离**：均值更新与欧氏距离天然匹配（最小化组内欧氏平方和）。若改用曼哈顿距离，应改为取中位数。

### 6.2 层次聚类

自底向上（凝聚式）或自顶向下（分裂式）构造树状结构，不需要预先给定 k，但计算开销大。

## 七、降维

### 7.1 PCA（主成分分析）

**目标**：找到方差最大的投影方向，在低维保留最多信息。

**实现**：对中心化数据的协方差矩阵做特征分解，取最大特征值对应的特征向量。等价于对数据矩阵做 SVD。

**局限**：线性方法，不一定保留最有预测价值的特征（只保留大方差方向）。

**神经网络隐层 vs PCA**：神经网络隐层关注任务相关表示，可能保留小方差但高信息量的特征，不等同于 PCA。

## 八、优化方法

### 8.1 梯度下降变体

| 方法 | 更新粒度 | 特点 |
|---|---|---|
| GD（全批量） | 全部样本 | 稳定但慢，内存压力大 |
| SGD（随机） | 单样本 | 快但噪声大，可在线学习 |
| Mini-batch SGD | 小批量 | 兼顾速度和稳定性 |

**梯度下降是局部最陡方向**，不是全局最快收敛路径。学习率过大会震荡，过小会收敛慢。

**牛顿法 vs 梯度下降**：牛顿法用二阶信息（Hessian 矩阵），收敛更快，但计算和存储 Hessian 代价极高（$O(n^2)$ 存储）。

### 8.2 梯度问题及解法

| 问题 | 根本原因 | 解法 |
|---|---|---|
| 梯度消失 | 深层反向传播时梯度连乘趋零 | ReLU / LSTM / 残差连接 / BatchNorm |
| 梯度爆炸 | 梯度连乘趋于无穷 | 梯度裁剪 / 合理初始化 |

## 九、模型评估

### 9.1 分类指标

$$\text{Precision} = \frac{TP}{TP+FP}, \quad \text{Recall} = \frac{TP}{TP+FN}$$

$$\text{F1} = \frac{2 \cdot P \cdot R}{P + R}$$

- **AUC**：衡量分类器整体排序能力，不依赖单一阈值，对类别不平衡更鲁棒
- **Precision vs Recall**：查准率和查全率通常是对立的，根据业务代价决定偏重

### 9.2 过拟合与欠拟合

| | 过拟合 | 欠拟合 |
|---|---|---|
| 表现 | 训练好，测试差 | 训练和测试都差 |
| 原因 | 模型太复杂 / 数据太少 | 模型太简单 / 特征不够 |
| 解法 | 正则化 / Dropout / 增加数据 / 早停 | 增加特征 / 提高复杂度 / 延长训练 |

### 9.3 类别不平衡

- 重采样（过采样少数类 / 欠采样多数类）
- 调整类别权重（class_weight）
- 改阈值（默认 0.5 不一定最优）
- 换指标：用 F1 / AUC 替代准确率
- Focal Loss：对难分样本加大权重

## 十、特征工程

**为什么特征工程比换模型更重要**：数据决定上限，特征决定模型是否能学到有效模式。好特征让简单模型也能很好地工作。

**主要操作**：

| 类型 | 操作 |
|---|---|
| 缺失值 | 均值/中位数填充 / 单独标记 |
| 异常值 | 截断 / 删除 / 分箱 |
| 归一化 | Min-Max / Z-score（LR、SVM、KNN 需要，树模型不需要）|
| 编码 | One-Hot / Label Encoding / Target Encoding |
| 特征选择 | 过滤式（相关系数）/ 包裹式（RFE）/ 嵌入式（L1 / 树重要性）|

**哪些模型对归一化敏感**：LR、SVM、KNN、KMeans、神经网络。决策树、随机森林、GBDT 不需要归一化（按阈值分裂，不依赖距离和梯度尺度）。

## 十一、完整项目流程

1. **问题定义**：分类 / 回归 / 排序 / 聚类，明确业务指标
2. **数据探查**：分布、缺失、异常、标签质量
3. **特征工程**：清洗 → 构造 → 筛选 → 归一化
4. **模型训练**：先用基线，再做复杂模型，交叉验证调参
5. **误差分析**：模型在哪些样本上错了，是数据问题还是特征问题
6. **模型融合**：Bagging / Boosting / Stacking
7. **上线与监控**：推理延迟、数据漂移、线上效果衰减

> 真正的项目不是一次性流程，而是持续迭代的闭环。

## 十二、深度学习基础

### CNN 适用范围

CNN 核心能力（局部感受野 + 参数共享 + 平移不变性）不局限于图像，适用于所有存在局部结构的数据：文本（1D 卷积）、语音、棋盘状态。

### LSTM vs RNN

RNN 存在梯度消失问题，长序列信息难以保留。LSTM 通过输入门、遗忘门、输出门和 cell state 明确控制信息的保留和遗忘，有效缓解长距离依赖问题。

### 归一化层

| 方法 | 归一化维度 | 适用场景 |
|---|---|---|
| BatchNorm | Batch 维度 | CNN，序列长度固定 |
| LayerNorm | 特征维度 | Transformer，变长序列 |
| GroupNorm | Group 维度 | 小 batch 场景 |
