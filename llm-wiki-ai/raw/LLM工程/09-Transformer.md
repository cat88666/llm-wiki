## 一、基础原理类

### 1. Transformer 核心数学模型：自注意力机制

Transformer 把输入序列表示成一组向量，然后通过注意力权重对这些向量进行加权组合，得到新的上下文化表示。对语言建模来说，本质是在学习条件概率：

- 给定上下文，预测下一个 token

`Attention(Q, K, V) = softmax(QK^T / sqrt(d_k)) V`

- 用 Query 和 Key 计算相关性
- 用 softmax 把相关性变成权重
- 用这些权重对 Value 做加权求和


Q、K、V 都是由输入表示经过线性变换得到的。

如果输入是 `X`，则：

- `Q = XW_Q`
- `K = XW_K`
- `V = XW_V`

`W_Q`、`W_K`、`W_V` 是训练出来的参数矩阵。

### 3. 为什么不能只保留 V，不用 Q 和 K

因为注意力包含两个问题：

- 谁和谁相关
- 聚合什么内容

如果只保留 V，就没有一套单独的机制来计算“匹配关系”，模型就很难灵活地区分：

- 当前 token 想查找什么
- 其他 token 能提供什么

Q/K 的存在，相当于把“检索”与“内容”解耦。

### 4. Self-Attention 为什么能表示长距离依赖

因为任意两个位置之间的交互路径长度是 1。

RNN 中，远距离依赖要经过多步状态传递，路径长，信息容易衰减。Transformer 中，一个 token 直接和全序列其他 token 建立注意力连接，因此更容易捕获远距离依赖。

### 5. 为什么说 Transformer 具有强表示能力

- Self-Attention 能建模全局依赖
- Multi-Head 能从多个子空间建模关系
- FFN 增强非线性特征变换
- 位置编码补充顺序信息
- 深层堆叠提升抽象表达能力

### 6. Feed Forward 层到底学到了什么

FFN 不是简单的“补一个线性层”，它学习的是：

- 在每个位置上的非线性特征变换
- 对 attention 融合后的表示再加工
- 为下游任务提取更适合判别或生成的局部特征

Attention 更偏“信息交互”，FFN 更偏“特征变换”。

### 7. Transformer 的神经网络为什么能很好地表示信息
- Self-Attention 负责跨位置建模
- 多头注意力让模型在多个子空间并行建模
- 位置编码引入顺序信息
- 残差连接和归一化让深层训练稳定
- 多层堆叠使模型具备多层级抽象能力

## 二、复杂度与计算类

### 8. Transformer 的 Embedding、Attention、FFN 复杂度分别是什么

- `n` 是序列长度
- `d` 是模型维度
- `h` 是头数
- `d_k` 是每个头的维度
- `d_ff` 是 FFN 隐藏层维度

常见分析：

- Embedding 查表本身代价不高，更多是内存访问问题
- Self-Attention 的核心复杂度来自 `QK^T`，通常是 `O(n^2 d)`
- FFN 通常是 `O(n d d_ff)`

结论：

- 长序列时，Attention 更容易成为瓶颈
- 模型维度很大时，FFN 也非常重

### 9. Multi-Head Attention 的复杂度会不会更高

从总量上看，不一定显著增加，因为多头通常是把总维度拆开。

例如：

- 总维度 512
- 8 个头
- 每个头 64 维

本质上是：

- 把同样的总表示能力分配到多个子空间

所以多头注意力更像是“表示方式变化”，而不是简单地把计算量乘以头数。

### 10. Self-Attention 为什么是 `O(n^2)`

因为每个位置都要和其他所有位置计算相关性。

如果序列长度是 `n`，注意力分数矩阵大小就是：

- `n × n`

所以随着序列变长，时间和显存都会快速上升。

### 11. 如何把 Self-Attention 从 `O(n^2)` 降到近似 `O(n)`

- 局部注意力：只看窗口内邻居
- 稀疏注意力：只保留部分连接
- 线性注意力：改写 softmax attention 形式
- 低秩近似：对 attention 矩阵做压缩
- 多尺度注意力：局部加少量全局

更工程化一点的回答：

- 长文档任务不一定必须全局 attention
- 先明确业务是否真需要全局依赖
- 再根据精度和吞吐要求做稀疏化或局部化

### 12. 为什么要除以 `sqrt(d_k)`

如果 `d_k` 很大，Q 和 K 的点积期望会变大，softmax 很容易进入饱和区间，导致：

- 梯度过小
- 训练不稳定

除以 `sqrt(d_k)` 的目的是控制数值尺度，使 softmax 更稳定。

## 三、位置编码类

### 13. 位置编码为什么必须存在

因为 Self-Attention 本身不关心顺序。

如果没有位置编码，模型只知道：

- 有哪些 token

但不知道：

- 它们出现的先后顺序
- 它们相隔多远

### 14. 正弦余弦位置编码如何表达相对位置关系

它通过不同频率的 `sin` 和 `cos` 函数给每个位置分配一个固定向量。

优点在于：

- 不同位置编码不同
- 编码有周期性
- 位置之间的差值可以通过三角函数关系部分体现出来

这使模型既能区分绝对位置，也能一定程度上学习相对距离。

### 15. 位置信息在多层 Encoder 之间会丢失吗

标准回答是：

- 不会直接“消失”
- 因为初始位置编码已经和 token embedding 融合进输入表示
- 这些表示会随着网络层层传播

但更严谨地说：

- 位置信息会被不断变换
- 不一定以“原始显式形式”保留
- 而是以网络内部表示的一部分继续存在

### 16. 位置编码有哪些替代方案

常见包括：

- 可学习位置编码
- 相对位置编码
- RoPE
- ALiBi

中如果扩展到大模型，可以补充：

- 长上下文任务里，相对位置编码和 RoPE 更常见

## 四、归一化与残差类

### 17. 为什么 Transformer 用 LayerNorm 而不是 BatchNorm

因为 Transformer 处理的是序列位置上的特征，每个 token 的表示更适合在特征维度上做归一化，而不是依赖 batch 统计量。

BatchNorm 在 NLP 场景中不如 CNN 中自然，原因包括：

- 序列长度变化大
- batch 统计不稳定
- 位置间依赖结构不同

### 18. LayerNorm 蕴含了什么假设

严格说它并不是“输入必须独立同分布”的强假设，而是一种数值稳定策略：

- 假设对单个样本内部特征做标准化有助于训练
- 让不同层的输入分布更稳定

如果面试官问这个问题，建议不要过度学术化，回答重点放在：

- 稳定训练
- 加快收敛
- 不依赖 batch 统计

### 19. 除了 LayerNorm，还有哪些归一化实现

在 Transformer 或大模型里常见的替代方案有：

- RMSNorm
- DeepNorm
- ScaleNorm

而 InstanceNorm、GroupNorm 更常见于视觉模型，不是 Transformer 主流答案。

### 20. 残差连接为什么重要

因为深层网络训练容易出现：

- 梯度消失
- 优化困难
- 表示退化

残差连接给了模型一条直接的信息通路，让每一层不必“从零学起”，有利于训练更深网络。

### 21. 如果 Transformer 层数特别深，比如 512 层，会出现什么问题

常见现象：

- 训练不稳定
- 梯度传播困难
- 收敛慢
- 显存和计算成本极高
- 过拟合风险上升

这也是为什么现代大模型除了堆层，还要配合：

- 更稳的归一化
- 更好的初始化
- 学习率 warmup
- 残差缩放

## 五、Encoder / Decoder 结构类

### 22. Encoder 和 Decoder 的数据依存关系是什么

Encoder 的作用是：

- 把输入序列编码成上下文化表示

Decoder 的作用是：

- 基于已生成前缀和 Encoder 输出，逐步生成目标序列

两者之间的依赖核心体现在：

- Decoder 的 cross-attention 依赖 Encoder 输出

所以从数据流角度：

- Encoder 先产出 memory
- Decoder 再基于这份 memory 做条件生成

### 23. Encoder 和 Decoder 的 Attention 有哪些不同

至少可以答 3 点：

#### 23.1 输入来源不同

- Encoder Self-Attention 的 Q/K/V 都来自输入序列本身
- Decoder Self-Attention 的 Q/K/V 来自目标侧历史
- Cross-Attention 中 Q 来自 Decoder，K/V 来自 Encoder

#### 23.2 是否需要因果 mask

- Encoder 通常不需要 sequence mask
- Decoder Self-Attention 必须屏蔽未来信息

#### 23.3 建模目标不同

- Encoder 更偏表示学习
- Decoder 更偏条件生成

### 24. 为什么 Decoder 中要有两种 Attention

因为它同时需要处理两类信息：

- 已经生成的目标端历史
- 源序列的编码结果

所以：

- Masked Self-Attention 负责建模目标前缀
- Encoder-Decoder Attention 负责读取源端信息

### 25. 如果训练时采用和推理完全一样的自回归流程，会有什么问题

会遇到两个核心问题：

- 训练效率很低，无法并行
- 训练分布和推理分布差异会暴露更多问题

常见术语包括：

- Teacher Forcing
- Exposure Bias

训练时通常喂真实前缀以提升效率和稳定性；推理时则只能用模型自己生成的历史 token。

### 26. 什么是 Exposure Bias

训练时模型看到的前缀通常是真实标签，推理时模型看到的前缀是自己生成的结果。

一旦某一步生成错了，后续输入分布就偏离训练分布，这就是暴露偏差。

## 六、Embedding 与 Tokenization 类

### 27. Tokenization 的原理、流程、问题是什么

核心流程：

1. 构建词表
2. 对文本切分
3. 映射成 token id
4. 查 embedding

主要问题：

- OOV
- 词表太大
- 跨语言一致性差
- 子词切得过细或过粗

改进思路：

- BPE
- WordPiece
- SentencePiece
- 动态词表
- 共享子词单元

### 28. BPE 对跨语言模型有什么意义

优点：

- 能把不同语言拆成共享子词单位
- 减少 OOV
- 提高稀缺语言泛化能力

问题：

- 不同语料学出来的词表未必一致
- 某些语言会被切得过碎

改进方法：

- 多语言联合训练词表
- 共享子词空间
- 多粒度表示

### 29. 输入 Embedding 时能否融合多来源词嵌入

可以。

常见做法：

- 直接拼接
- 平均融合
- 加权融合
- 通用 embedding + 领域 embedding 融合

好处：

- 融合多语料、多领域知识

风险：

- 维度不一致
- 训练不稳定
- 参数量增加

### 30. 如何降低 Embedding 层参数量

常见方法：

- 减小 embedding 维度
- 词嵌入矩阵共享
- 输入输出 embedding tying
- 子词建模减少词表
- 哈希 embedding

一句话总结：

- Embedding 参数量主要由“词表大小 × embedding 维度”决定

## 七、CLS 与句向量类

### 31. BERT 的 CLS 能很好表示句向量吗

标准回答应当是：

- 有时可以
- 但不能一概而论

原因：

- CLS 在分类任务微调后往往有效
- 但直接拿预训练 CLS 做通用 sentence embedding，效果未必最好

更常见的改进方案：

- mean pooling
- contrastive learning
- Sentence-BERT 风格训练

### 32. 为什么训练后的 BERT 不一定容易泛化

常见原因：

- 预训练任务和下游任务分布不一致
- 固定词表对新领域 OOV 不友好
- 微调可能造成灾难性遗忘
- CLS 并不天然就是最优句向量

## 八、训练策略类

### 33. 更深更宽的 Transformer 一定更强吗

不一定。

要从 3 个角度回答：

#### 33.1 参数表达能力

- 更深更宽通常意味着更强拟合能力

#### 33.2 工程落地

- 训练成本更高
- 显存压力更大
- 通信更复杂

#### 33.3 数据和任务匹配

- 模型变大前提是数据和训练策略也要跟上
- 否则收益会递减甚至过拟合

### 34. Batch 大小和困惑度 PPL 有什么关系

直观上：

- 大 batch 梯度更稳定
- 小 batch 噪声更大

但这不意味着 batch 越大越好。

真正影响 PPL 的是：

- 有效 batch
- 学习率策略
- token 总训练量
- 优化器配置

面试中不要回答成绝对单调关系。

### 35. 小样本、多类别任务怎么训练 Transformer

常用思路：

- 类别加权 loss
- 分层采样
- 数据增强
- 迁移学习
- few-shot prompt 或 instruction tuning
- 原型网络或 metric learning 思路

### 36. 类别极不平衡怎么办

可答组合拳：

- 加权交叉熵
- focal loss
- 重采样
- 分层 batch
- 大类欠采样、小类过采样
- 利用预训练再微调

### 37. 多类别小样本如何取得更好分类效果

如果面试官追问“背后机制”，可以答：

- 用预训练模型提供通用特征
- 通过类别嵌入或原型表示增强类别先验
- 通过注意力机制对少样本特征做动态聚合
- 通过加权损失缓解类别偏斜

## 九、Mask 与张量形状类

### 38. 如何对 Transformer 任意位置和长度实现 Mask

本质是构造一个 mask 矩阵，与 attention score 同形或可广播。

常见做法：

- 允许的位置设为 0
- 禁止的位置加上一个极小值，如 `-inf`

然后在 softmax 前把它加到 attention score 上。

这样被 mask 的位置 softmax 后概率接近 0。

### 39. 为什么 Transformer 的输入张量常写成 3D

因为通常有三个维度：

- batch size
- sequence length
- hidden size

也就是常见形状：

- `[B, N, D]`

其中：

- `B` 决定并行样本数
- `N` 决定序列长度与 attention 开销
- `D` 决定表示维度与模型容量

### 40. 每个维度变化会带来什么影响

- `B` 增大：吞吐提升，但显存和优化动态变化
- `N` 增大：attention 二次增长，显存压力大
- `D` 增大：参数量和矩阵乘法成本上升

## 十、Attention 扩展理解类

### 41. 为什么说注意力机制相对“便宜”

这个说法要谨慎。

如果和 RNN 比：

- Transformer 训练更易并行

如果和长序列自己比：

- 标准 attention 其实并不便宜

更准确的回答应该是：

- 它在并行训练和全局依赖建模上更高效
- 但在长序列场景下计算和显存成本很高

### 42. 多头注意力为什么有效

因为不同头可以在不同子空间关注不同类型的信息，比如：

- 指代
- 句法依赖
- 局部搭配
- 远距离语义关系

最后拼接融合后，得到更丰富表示。

### 43. 如何判断 Transformer 中神经元是否重要

常见方法：

- 基于梯度大小
- 基于激活值大小
- 基于剪枝后的性能下降
- 基于信息熵或贡献度分析

工程上真正有用的是：

- 看删掉后性能掉多少

## 十一、句向量、相似度与迁移类

### 44. 如何比较不同模型产生的 Embedding 相似度

常见方法：

- 余弦相似度
- 欧氏距离
- 点积

如果维度分布差异明显，通常优先用：

- cosine similarity

### 45. 如何把大模型 Embedding 迁移给小模型

常见方式：

- 参数初始化
- 特征蒸馏
- 软标签蒸馏
- 中间层表示对齐

### 46. Zero-shot Learning 如何用 Transformer 实现

核心思想：

- 让类别标签或类别描述也表示成向量
- 把样本表示和类别语义表示放到同一空间比较

这样即使新类别没见过，只要有文本描述，模型也能基于语义相似性做判断。

## 十二、长序列、加速与压缩类

### 47. 至少说三种提升 Transformer 预测速度的方法

常见答案：

- 减少层数
- 减少头数
- 降低 hidden size
- KV cache
- 蒸馏
- 量化
- 剪枝
- 更高效 attention kernel

如果要带数学味道，可以补充：

- 本质上是在减少矩阵乘法规模、注意力开销或显存访存成本

### 48. 如何降低 FFN 层参数量

FFN 参数量主要来自：

- `d_model × d_ff`
- `d_ff × d_model`

降低方法：

- 减小 `d_ff`
- 低秩分解
- 参数共享
- 稀疏化
- MoE 式条件激活

### 49. 为什么 FFN 是参数大户

因为 attention 虽然复杂，但 FFN 在每层通常包含两个大矩阵，参数量往往很大。

很多大模型里：

- 参数主要集中在 embedding 和 FFN

### 50. 为什么工业量化通常要基于校准集或重新训练

因为量化会改变数值分布。

如果不做校准：

- 激活范围估计不准
- 精度损失可能很大

所以通常需要：

- calibration
- quantization-aware training

### 51. 为什么量化能提速

因为：

- 权重更小
- 带宽开销更低
- 某些硬件对低精度计算更高效

但不是所有场景都必然线性加速，还要看：

- kernel 实现
- 硬件支持
- memory bound 还是 compute bound

## 十三、预训练任务类

### 52. NSP 任务有什么问题

常见问题：

- 负样本构造简单
- 任务和真实语义理解未必高度一致
- 训练开销增加

这也是后来很多 BERT 变体弱化或移除了 NSP 的原因。

### 53. 为什么随机 Mask 任务也有缺陷

因为它会带来：

- 预训练和下游不一致
- [MASK] token 在下游通常不存在
- 某些 token 可从局部上下文轻易猜出，任务过于简单

### 54. 如何改进 MLM 训练

常见方向：

- 更难的 mask 策略
- span mask
- whole word mask
- 结合句子级任务
- 替换为更符合生成任务的目标

## 十四、泛化与理论类

### 55. 神经网络为什么能表示任意复杂度的信息

理论上可从通用逼近定理回答：

- 足够宽或足够深的神经网络可以逼近大量连续函数

但面试里不要只背定理，更好的回答是：

- 线性层负责投影
- 非线性激活提供函数族扩展
- 深层堆叠形成分层抽象表示

### 56. 为什么 Transformer 不一定容易泛化

原因包括：

- 预训练分布和下游分布有差异
- tokenization 和词表限制
- 微调造成遗忘
- 模型可能依赖表层模式而非稳健语义

### 57. 为什么教师强制会带来训练偏差

因为训练时每一步都喂正确前缀，模型没有真正学会：

- 如何在自己出错后继续恢复

这会导致训练分布与推理分布不一致。

## 十五、回答质量提升类

### 58. Transformer 面试题回答时最容易犯的错误是什么

常见错误：

- 只背定义，不讲为什么
- 公式会写，但不会解释工程含义
- 把 LayerNorm、BatchNorm、RMSNorm 混为一谈
- 把 Attention 说成万能，但说不清复杂度代价
- 一谈长序列优化就只会说“局部 attention”

### 59. 回答原理题的推荐结构是什么

建议统一用四步：

1. 先给定义
2. 再给公式或机制
3. 再讲优缺点
4. 最后落到工程意义

例如回答“为什么要多头注意力”：

- 定义：多个头并行做 attention
- 机制：把表示映射到多个子空间
- 优点：学习不同关系
- 工程意义：在总维度近似不变下提升表达力

## 十六、高频速答清单

### 60. 为什么要 Q/K/V 三个矩阵

- Q/K 算相关性
- V 承载内容
- 解耦匹配与聚合

### 61. 为什么要位置编码

- Attention 本身不感知顺序

### 62. 为什么用 LayerNorm

- 不依赖 batch 统计
- 更适合序列建模

### 63. 为什么用残差

- 让深层训练更稳定

### 64. 为什么 Transformer 训练快

- 可并行

### 65. 为什么 Transformer 推理不一定快

- 自回归生成仍然串行

### 66. 为什么长序列难

- attention 是 `n^2`

### 67. 为什么 CLS 不一定是最佳句向量

- 它是为预训练目标服务的，不一定天然适合语义相似任务

### 68. 为什么大模型不一定泛化更好

- 参数变大不等于数据、优化和任务都匹配

## 十七、延伸问答

### 69. 如果面试官问“Transformer 为什么不天然适合超长文本”

答：

- 标准 self-attention 的时间和显存复杂度都随序列长度二次增长
- 因此长文本场景必须考虑稀疏化、分块、缓存或检索增强

### 70. 如果面试官问“Decoder 为什么要 Mask”

答：

- 为了保证自回归约束
- 当前位置不能看到未来 token

### 71. 如果面试官问“Embedding 能不能完全替代 tokenization 问题”

答：

- 不能
- embedding 依赖 tokenization 结果
- tokenization 决定了基本离散单位

### 72. 如果问“为什么 Batch 大小会影响训练稳定性”

答：

- batch 决定梯度估计方差
- 小 batch 噪声更大
- 大 batch 更稳，但泛化和优化路径不一定更优

### 73. 如果问“为什么 FFN 对每个位置独立”

答：

- 跨位置交互已经在 attention 中完成
- FFN 负责对每个位置做通道方向上的非线性变换

### 74. 如果问“Transformer 里参数最多的部分通常在哪”

答：

- 常见在 embedding 和 FFN
- attention 更像是计算瓶颈，FFN 更常是参数瓶颈之一

## 十八、复习建议

1. 第一轮：Q/K/V、Attention、Multi-Head、位置编码
2. 第二轮：Encoder / Decoder、mask、训练推理差异
3. 第三轮：复杂度、长序列优化、Tokenization、CLS
4. 第四轮：压缩、量化、小样本、泛化、工程问题
