# query 技能

## 核心目标

把用户问题转化为对 `llm-wiki-ai` 的结构化检索，并基于已有 wiki 页面回答。回答有稳定沉淀价值时，可回写到知识库。

## 触发场景

- 用户输入 `/query <问题>`
- 用户询问"我的知识库里怎么说""之前笔记里怎么总结"
- 用户明确要求基于本地 wiki 回答

## 检索与综合流水线

### 步骤 1：先读全局索引

第一步必须读取 `llm-wiki-ai/index.md`，从索引中定位相关页面。

不要引用不存在的 `wiki/sources/` 或 `wiki/syntheses/` 目录。

### 步骤 2：深度阅读目标页面

读取最相关页面的完整内容，必要时交叉读取多个页面，为关键论点标注来源路径或页面名。

### 步骤 3：综合回答

- 优先基于本地知识库内容回答
- 若本地知识库没有足够信息，明确说明"本地知识库中未找到充分内容"，再给出通用回答
- 不要把通用知识伪装成来自本地 wiki 的结论

### 步骤 4：高价值内容回写

如果回答本身形成稳定知识，按以下原则决定是否回写：

- 属于某个知识主题的稳定总结 → 更新 `wiki/summaries/`
- 属于跨主题判断、比较、策略 → 更新 `wiki/synthesis/`
- 属于概念、机制、方法或抽象原理 → 更新 `wiki/concepts/`

只有在确实需要新增页面时才新建。新建页面前先按 `naming/SKILL.md` 确定文件名（铁律3）。

### 步骤 5：记录操作日志（铁律2）

查询结束后追加 `llm-wiki-ai/log.md`。如有新增或回写内容，同步更新 `llm-wiki-ai/index.md`。

## 强制约束

- 禁止跳过检索直接回答
- 禁止引用当前仓库不存在的目录结构
- 若回写知识，必须区分 `summary` 和 `synthesis`
- 若回写 `concepts`，优先写成原理型页面，结构见根 `CLAUDE.md` 铁律5
